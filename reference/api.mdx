RAG-Anything exposes a layered public API. The `RAGAnything` class is the primary entry point for most use cases. Lower-level classes — modal processors, `ContextExtractor`, `BatchParser` — are available for advanced integrations. Callbacks, prompt management, and resilience decorators are optional add-ons imported from the top-level `raganything` package.

---

## RAGAnything

`RAGAnything` is a dataclass that composes three mixins — `ProcessorMixin`, `BatchMixin`, and `QueryMixin` — providing the complete document-processing-to-query pipeline.

```python
from raganything import RAGAnything, RAGAnythingConfig
```

### Constructor

<AccordionGroup>
  <Accordion title="RAGAnything(...)">
    ```python
    @dataclass
    class RAGAnything(QueryMixin, ProcessorMixin, BatchMixin):
        lightrag: Optional[LightRAG] = None
        llm_model_func: Optional[Callable] = None
        vision_model_func: Optional[Callable] = None
        embedding_func: Optional[Callable] = None
        config: Optional[RAGAnythingConfig] = None
        lightrag_kwargs: Dict[str, Any] = field(default_factory=dict)
    ```

    <ParamField path="lightrag" type="LightRAG | None" default="None">
      Pre-initialised `LightRAG` instance. When provided, `llm_model_func` and `embedding_func` are inherited from it and `lightrag_kwargs` is ignored.
    </ParamField>

    <ParamField path="llm_model_func" type="Callable | None" default="None">
      LLM callable used for text analysis and entity extraction. Required when `lightrag` is not provided.
    </ParamField>

    <ParamField path="vision_model_func" type="Callable | None" default="None">
      Vision-language model callable for image captioning and VLM-enhanced queries. When omitted, image processing falls back to the LLM.
    </ParamField>

    <ParamField path="embedding_func" type="Callable | None" default="None">
      Embedding callable for text vectorisation. Required when `lightrag` is not provided.
    </ParamField>

    <ParamField path="config" type="RAGAnythingConfig | None" default="None">
      Configuration object. When `None`, a default `RAGAnythingConfig()` is constructed, which reads from environment variables. See the [configuration reference](/reference/configuration).
    </ParamField>

    <ParamField path="lightrag_kwargs" type="Dict[str, Any]" default="{}">
      Extra keyword arguments forwarded verbatim to `LightRAG(...)` when RAGAnything creates its own LightRAG instance. Common keys include `kv_storage`, `vector_storage`, `graph_storage`, `top_k`, `chunk_token_size`, `embedding_batch_num`, `max_parallel_insert`, and `enable_llm_cache`.
    </ParamField>
  </Accordion>
</AccordionGroup>

    </ParamField>

    <ParamField path="vlm_enhanced" type="bool">
      Override VLM behaviour. `True` forces VLM processing; `False` disables it. Defaults to `True` when `vision_model_func` is available.
    </ParamField>

    <ResponseField name="return" type="str">
      Generated answer from the LLM.

## Modal processors

All processors inherit from `BaseModalProcessor` and share the same constructor signature and `process_multimodal_content` interface.

```python
from raganything.modalprocessors import (
    ImageModalProcessor,
    TableModalProcessor,
    EquationModalProcessor,
    GenericModalProcessor,
)
```


    The same signature applies to `TableModalProcessor`, `EquationModalProcessor`, and `GenericModalProcessor`.

    </ParamField>

    <ParamField path="context_extractor" type="ContextExtractor | None" default="None">
      Custom `ContextExtractor` instance. When omitted, a default extractor is created using the tokenizer from the `LightRAG` instance.
    </ParamField>

    </ParamField>

    <ResponseField name="return" type="Tuple[str, Dict, List]">

    Subclass and override any `on_*` method to hook into pipeline stages. All methods accept `**kwargs` for forward compatibility.

    | Method | Trigger |
    |--------|---------|
    | `on_parse_start(file_path, parser, **kw)` | Before document parsing begins |
    | `on_parse_complete(file_path, content_blocks, doc_id, duration_seconds, **kw)` | After successful parsing |
    | `on_parse_error(file_path, error, **kw)` | When parsing fails |
    | `on_text_insert_start(file_path, text_length, **kw)` | Before text insertion into LightRAG |
    | `on_text_insert_complete(file_path, duration_seconds, **kw)` | After text insertion succeeds |
    | `on_multimodal_start(file_path, item_count, **kw)` | Before multimodal processing begins |
    | `on_multimodal_item_complete(file_path, item_index, item_type, total_items, **kw)` | After each multimodal item |
    | `on_multimodal_complete(file_path, processed_count, duration_seconds, **kw)` | After all multimodal items |
    | `on_query_start(query, mode, **kw)` | Before query execution |
    | `on_query_complete(query, mode, duration_seconds, result_length, **kw)` | After query completes |
    | `on_query_error(query, mode, error, **kw)` | When a query fails |
    | `on_document_complete(file_path, doc_id, duration_seconds, **kw)` | When full pipeline finishes |
    | `on_document_error(file_path, error, stage, **kw)` | When pipeline fails at any stage |
    | `on_batch_start(file_count, **kw)` | When batch processing begins |
    | `on_batch_complete(total_files, successful, failed, duration_seconds, **kw)` | When batch processing finishes |

    ```python
    class MyCallback(ProcessingCallback):
        def on_parse_complete(self, file_path, content_blocks=0, **kw):
            print(f"Parsed {content_blocks} blocks from {file_path}")

    rag.callback_manager.register(MyCallback())
    ```
  </Accordion>

  <Accordion title="MetricsCallback">
    Built-in callback that accumulates processing statistics. Access via the `metrics` dict or `summary()`.

    ```python
    from raganything import MetricsCallback

    cb = MetricsCallback()
    rag.callback_manager.register(cb)

    # After processing:
    print(cb.summary())
    # RAGAnything Processing Metrics
    # ========================================
    # Documents processed : 5
    # Documents failed    : 0
    # Content blocks      : 312
    # ...

    cb.reset()  # Clear all counters
    ```
  </Accordion>

  <Accordion title="CallbackManager">
    Thread-safe dispatcher owned by every `RAGAnything` instance as `rag.callback_manager`.

    | Method | Description |
    |--------|-------------|
    | `register(callback)` | Add a `ProcessingCallback` instance |
    | `unregister(callback)` | Remove a registered callback |
    | `dispatch(event_name, **kwargs)` | Fire an event (used internally by RAGAnything) |
    | `enable_event_log(enabled)` | Start/stop recording `ProcessingEvent` objects |
    | `event_log` | Read-only list of recorded `ProcessingEvent` objects |
    | `clear_event_log()` | Clear the event log |
  </Accordion>

  <Accordion title="ProcessingEvent">
    Immutable record of a pipeline event with fields: `event_type`, `timestamp`, `file_path`, `doc_id`, `stage`, `details`, `duration_seconds`, `error`. Call `.to_dict()` to serialise.
  </Accordion>
</AccordionGroup>

## Resilience utilities

```python
from raganything import retry, async_retry, CircuitBreaker
```

<AccordionGroup>
  <Accordion title="@retry(...)">
    ```python
    def retry(
        max_attempts: int = 3,
        base_delay: float = 1.0,
        max_delay: float = 60.0,
        exponential_base: float = 2.0,
        jitter: bool = True,
        retryable_exceptions: Optional[Sequence[Type[BaseException]]] = None,
        on_retry: Optional[Callable[[BaseException, int, float], None]] = None,
    ) -> Callable
    ```

    Decorator for synchronous functions. Uses exponential back-off with jitter.

    ```python
    from raganything import retry

    @retry(max_attempts=5, base_delay=2.0)
    def call_external_api():
        ...
    ```

    <ParamField path="max_attempts" type="int" default="3">
      Total attempts including the first call.
    </ParamField>
    <ParamField path="base_delay" type="float" default="1.0">
      Initial sleep duration in seconds.
    </ParamField>
    <ParamField path="max_delay" type="float" default="60.0">
      Maximum sleep duration in seconds.
    </ParamField>
    <ParamField path="retryable_exceptions" type="Sequence[Type[BaseException]] | None" default="None">
      Defaults to common network and API transient errors (`ConnectionError`, `TimeoutError`, `httpx` errors, `openai` errors when installed).
    </ParamField>
  </Accordion>

  <Accordion title="@async_retry(...)">
    ```python
    def async_retry(
        max_attempts: int = 3,
        base_delay: float = 1.0,
        max_delay: float = 60.0,
        exponential_base: float = 2.0,
        jitter: bool = True,
        retryable_exceptions: Optional[Sequence[Type[BaseException]]] = None,
        on_retry: Optional[Callable] = None,
    ) -> Callable
    ```

    Async counterpart of `@retry`. Uses `asyncio.sleep` so it does not block the event loop.

    ```python
    from raganything import async_retry

    @async_retry(max_attempts=5, base_delay=2.0)
    async def call_llm_async(prompt: str) -> str:
        ...
    ```
  </Accordion>

  <Accordion title="CircuitBreaker">
    ```python
    class CircuitBreaker:
        def __init__(
            self,
            failure_threshold: int = 5,
            reset_timeout: float = 60.0,
            name: str = "default",
            failure_exceptions: Optional[Sequence[Type[BaseException]]] = None,
        )
    ```

    Prevents cascading failures. Opens after `failure_threshold` consecutive failures; enters half-open state after `reset_timeout` seconds.

    ```python
    from raganything import CircuitBreaker

    breaker = CircuitBreaker(failure_threshold=3, reset_timeout=30.0, name="llm")

    @breaker          # sync function
    def call_llm(prompt):
        ...

    @breaker.async_call   # async function
    async def call_llm_async(prompt):
        ...
    ```

    | Property / Method | Description |
    |-------------------|-------------|
    | `state` | Current state: `"closed"`, `"open"`, or `"half-open"` |
    | `record_success()` | Manually record a success and reset the breaker |
    | `record_failure()` | Manually record a failure, potentially opening the breaker |
    | `CircuitBreakerOpen` | Exception raised when the circuit is open |
  </Accordion>
</AccordionGroup>
